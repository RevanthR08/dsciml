# 🛡️ LogAI SOC Platform API Documentation

This project provides a **Research-Grade SOC Detection System** that combines Rule-based logic, Machine Learning (Isolation Forest), and AI Intelligence (Gemini/Llama) to analyze forensic logs.

The API is built using **FastAPI** and is designed to be consumed by external UI projects.

---

## 🚀 Getting Started

1. **Start the API Server**:
   ```powershell
   python main_api.py
   ```
2. **Base URL**: `http://localhost:8000`
3. **CORS Enabled**: The API is "Unlocked" and can be called from any frontend domain (React, Vue, HTML, etc.).

---

## 📡 API Endpoints

### 1. Home / Health Check
- **Endpoint**: `GET /`
- **What it does**: Verifies the server is online and lists available endpoints.
- **Use case**: Use this to check if your backend is "Up" before loading your UI dashboard.

---

### 2. Trigger Forensic Scan
- **Endpoint**: `POST /run-scan`
- **What it does**: 
  - Triggers the `forensic_report.py` script in a background thread.
  - Runs Rule-Based detection + ML Anomaly detection.
  - Generates a new timestamped JSON report in the `detected_anomalies/` folder.
- **Use case**: Connect this to your **"Scan Now"** or **"Upload"** button in your UI.

---

### 3. Fetch Latest Detection Data (JSON)
- **Endpoint**: `GET /latest-json`
- **What it does**: 
  - Finds the most recent JSON report generated by the ML engine.
  - Returns the full structured data: MITRE mappings, risk scores, and individual log events.
- **Use case**: Use this to populate **Charts, Graphs, and Data Tables** in your UI.

---

### 4. Fetch AI Executive Briefing
- **Endpoint**: `GET /ai-summary`
- **What it does**: 
  - Triggers the **AI Intelligence Engine**.
  - Uses a **Map-Reduce (Chunking)** strategy to analyze large logs.
  - Compiles a high-level report including "Critical Findings," "Attacker Strategy," and "Remediation Plan."
- **Use case**: Use this to show a **"Summary"** section or a **"Security Briefing"** panel.

---

### 5. Interactive Security Chat
- **Endpoint**: `POST /ask`
- **What it does**: 
  - Accepts a JSON body: `{"question": "Your question here"}`.
  - The AI looks into the latest forensic entries (with actual timestamps and computer names) to answer precisely.
- **Use case**: Use this for a **"Chat with your SOC Analyst"** feature in your UI.

---

## 🧩 Integration Example (JavaScript)

If you are using JavaScript to connect your UI, here is how you call the AI Chat:

```javascript
const response = await fetch('http://localhost:8000/ask', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ question: "Why was WORKSTATION-08 flagged?" })
});
const data = await response.json();
console.log("AI Answer:", data.answer);
```

## ⚙️ Environment Configuration (`.env`)
Make sure your API keys are set in the `.env` file:
- `GEMINI_API_KEY`: For Google Gemini 1.5/2.0 mode.
- `HF_API_TOKEN`: For Hugging Face (Llama 3.1) mode.

The system will automatically choose the best available backend!
